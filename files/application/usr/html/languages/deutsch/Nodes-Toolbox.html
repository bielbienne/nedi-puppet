Einführung

Dieses Modul wurde von Eli Gill entwickelt. Remo Rickli hat alle notwendigen Anpassungen vorgenommen, damit dieses Modul mit den unterschiedlichen Frontend-Ansichten (Themen), die NeDi zur Verfügung stellt, verwendet werden kann. Auf Wunsch von mir wurde von Remo auch eine Betriebsystemerkennung für die Windows-Plattform eingebaut. Dies ist notwendig, da der Ping-Bereich unter Windows mit diesem Modul (Fping) nicht funktioniert.

Benutzung

Die Benutzung dieses Moduls sollte weitgehend selbsterklärend sein. Wird die Funktion Traceroute für ein Ziel verwendet, welches nicht über dieses ICMP-Protokoll erreichbar ist (z.B. falsche Angabe, Firewall, etc.), dann muss, bis eine Fehlermeldung ausgegeben wird, mit einem ziemlich langen Timeout gerechnet werden.

Wer möchte, kann "Putty.exe" im NeDi-Verzeichnis (NeDi\html\log) ablegen und dieses nützliche Terminal-Programm über das Symbol dann von jedem Standort ausführen.

Wichtig ist hierbei, dass auf dem Webserver für das Verzeichnis (NeDi\html\log) keine Rechte für ausführbare Dateien gesetzt worden sind. Ansonsten wird versucht, die Datei "Putty.exe" auf dem Server auszuführen anstatt diese zum Download anzubieten.
