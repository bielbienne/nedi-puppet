Monitoring Meldungen

Einführung

In diesem Modul werden eingehende Syslog-Nachrichten und SNMP-Traps angezeigt, sofern die Geräte entsprechend konfiguriert worden sind. Mit Hilfe der vielen Filtermöglchkeiten können Probleme schnell und effizient eingegrenzt werden.

Benutzung

Mit dem Button "Zeigen" können die Nachrichten aufgelistet werden. Im Feld [Limite] kann die Anzahl der anzuzeigenden Nachrichten eingestellt werden. Wählt man dort über das Drop Down Menü den Eintrag <none!> um alle Nachrichten anzuzeigen, kann dies bei einer grossen Anzahl Nachrichten negative Auswirkungen auf die Perfomance von NeDi haben.

Die Felder [Bedingung A] und [Bedingung B] können genutzt werden, um die Abfragen zu filtern. Das Feld [Bedingung A] kann ohne das Feld [Bedingung B] genutzt werden. Soll das Feld [Bedingung B] ebenfalls verwendet werden, muss dies via dem Drop Down Menü im Feld [Verknüpfung] angegeben werden. In den Feldern [Bedingung A] und [Bedingung B] besteht nebst der Nutzung der Drop Down Menüs auch noch die Möglichkeit, die angezeigten Nachrichten durch Verwendung des jeweiligen Textfeldes zusätzlich zu filtern.

Via dem Drop Down Menü im Feld [Level] ist es möglich, innerhalb der angezeigten Nachrichten nur noch Nachrichten mit einem bestimmten Level anzuzeigen. Belässt man im Feld [Level] das Drop Down Menü auf dem Standardwert <Auswahl>, kann man stattdessen für das Feld [Bedingung A] vordefinierte Filter über die Symbole nutzen nutzen. Die angezeigten Nachrichten können bei Bedarf in einem zweiten Schritt via dem Feld [Bedinung B] zusätzlich gefiltert werden.

Unterhalb des Buttons "Zeigen", kann mit den Pfeilsymbolen innerhalb der Resultate geblättert werden. Der Button "Löschen" löscht die angezeigten Nachrichten permanent aus der Datenbank.

Klickt man in den aufgelisteten Nachrichten in der Spalte [Level] auf ein Icon, werden nur noch Nachrichten desselben Levels dargestellt. Dasselbe gilt für die Spalte [Quelle]. Klickt man auf einen Gerätenamen, werden nur noch Nachrichten bezogen auf dieses Gerät angezeigt.

Details

  • Um mit diesem Modul zu arbeiten, ist das Verstehen von Regular Expressions (Abk. RegExp oder Regex) hilfreich.
  • Abfragen können als Favoriten im Browser hinterlegt werden.
  • Am Ende der Tabellen steht immer die resultierende SQL-Abfrage. Diese kann z.B. im Modul {Other -> Export} weiter verwendet werden.
Die Bedeutung der Symbole im Feld [Level] zum Filtern der Syslog Meldungen:
  • Zeigt Meldungen von aufgetretenen Fehlern bei der Datenübertragung an.
  • Zeigt Meldungen bezogen auf die Gerätekonfigurationen an (z.B. Änderungen, Backupprobleme etc.).
  • Zeigt Meldungen von aufgetretenen Fehlern beim Netzwerk-Discovery an. Zeigt zusätzlich auch zu hohe Temperaturen, zu hohe Prozessorlast etc. von Geräten auf.
  • Zeigt Meldungen bezogen auf das Anmelden auf den Geräten an z.B. Loginfehler, Wechsel in den privilegierten Modus etc.
  • Zeigt Meldungen von neu gestarteten Geräten an.
  • Zeigt Host spezifische Meldungen an z.B. wenn ein als gestohlen markiertes Gerät wieder im Netzwerk entdeckt wird.
Die Bedeutung der Symbole in der Spalte [Action]:
  • Link zum Modul {Geräte Status} des jeweiligen Gerätes.
  • Link zum Modul {Geräte Konfiguration} des jeweiligen Gerätes.
  • Link zum Modul {Node Status} des jeweiligen Gerätes.
  • Link zum Modul {Node List} mit gesetztem Filter für das jeweilige Gerät.
