Topologie Karte

Einführung

Dieses Modul ermöglicht das Erstellen einer graphischen Netzwerkdarstellung und ist daher sehr nützlich, um das Netzwerk zu dokumentieren. Das Erstellen der Netzwerkkarte basiert auf den Netzwerkgeräten konfigurierten SNMP-Standortinformationen. Obwohl die Symbole in den erstellten Netzwerkkarten interaktiv sind und auf weitere NeDi Module verweisen, liegt der Schwerpunkt klar beim Abbilden des Netzwerks.

Damit dieses Modul korrekt funktioniert, muss der Webserver folgende zusätzliche Kriterien erfüllen:
  • Auf dem Webserver muss die PHP-GD-Erweiterung aktiviert sein.
Auf dem Webserver müssen folgenden MIME Types gesetzt sein, damit die Vektorgraphiken korrekt funktionieren:
  • Endung: .svg MIME: image/svg+xml
  • Endung: .svgz MIME: image/svg+xml
Ausserdem müssen die SNMP-Standortangaben auf den Netzwerkgeräten korrekt konfiguriert worden sein und auf dem Client muss eventuell noch der freie Adobe SVG Viewer installiert werden (Browser Plugin).

Netzwerkkarten werden pro Benutzer erstellt und im NeDi-Verzeichnis ...html\log abgelegt (Schreibrechte für den Webserver erforderlich). Solange an einer erstellten Netzwerkkarte nichts verändert wird, wird die erstellte Netzwerkgraphik bei einem späteren Besuch dieses Moduls wieder gleich dargestellt, jedoch ohne interaktive Funktionen.

Es besteht die Möglichkeit, Bandbreitengraphiken in die Links zu integrieren. Diese Graphiken werden als .png Dateien im NeDi-Verzeichnis ...html\log abgelegt. Diese .png Graphiken werden jeweils bei allen nachfolgenden Anpassungen an der Netzwerkkarte wieder überschrieben. Um also von einem bestimmten Zeitpunkt eine Netzwerkkarte zu haben, sollte die Karte manuell als .png Datei gespeichert werden (z.B. rechte Maustaste | Bild speichern unter). Alternativ kann natürlich auch einfach ein Screenshot gemacht werden. RRD's (Verkehrs/Fehlergraphiken) stehen bei Vektorgraphiken (SVG) nicht zur Verfügung.

Benutzung

  • Auf der NeDi-Homepage findet man ein Multimedia-Tutorial, welches den Einstieg erleichtert.
  • Geographische Karten werden im Modul {Topologie Loced} konfiguriert.
  • Mit dem Button "SVG" kann eine Vektorzeichnung erstellt werden, welche anschliessend in andere Programme importiert werden kann.
  • Fährt man mit der Maus über die Felder und Symbole, wird eine kontextbasierte Hilfe eingeblendet.

Um eine Netzwerkkarte zu erstellen, muss nach dem Auswählen der nachfolgend beschriebenen Optionen der Button "PNG" oder "SVG" angeklickt werden.

Feld Image:

  • Über das Drop Down Menü [Grösse] können die Abmessungen der Zeichnung anhand von vordefinierten Grössen festgelegt werden.
  • Via den Textfeldern [xy] können die Abmessungen der Netzwerkkarte selbst definiert werden.
  • Über die Auswahl [Tiefe] kann die Qualität und die Grösse der Netzwerkkarte beeinflusst werden. Der Wert "24bit" ist vordefiniert und sollte so belassen werden.

Feld Generell:

  • Über das Textfeld [Titel] kann die Überschrift der Netzwerkkarte festgelegt werden.
  • Über das linke Drop Down Menü kann die Detailebene der Karte bestimmt werden. Damit ist die Unterscheidung von Regionen, Ländern, Gebäuden etc. gemeint. Zusätzlich kann über das rechte Drop Down Menü der Zeichnungsmodus festgelegt werden. Im Geo-Modus können vordefinierte geographische Karten aus dem Modul {Topologie Loced} aktiviert werden.
  • Die Textfelder [Offset] ermöglichen die Positionierung der Netzwerkkarte anzupassen. Es kann auch mit negativen Werten gearbeitet werden. Der Offset-Wert "@" ist zum Rotieren von mehreren Ortschaften gedacht!

Feld Layout:

  • Über die Drop Down Menüs [Link] kann die Darstellung der Links angepasst werden. Es ist auch möglich, Verkehr, oder Fehler-Graphiken in die Links zu integrieren.
  • Über die Drop Down Menü [Point] können die verwendeteten Symbole ausgewählt werden. Über die zugehörigen Textfelder können die Links beinflusst werden.
  • Die Textfelder [Floor] ermöglichen es die Abmessungen der Stockwerke auf der Netzwerkkarte anzupassen. Es kann auch mit negativen Werten gearbeitet werden. Der Offset Wert "@" ist zum Rotieren von mehreren Gebäuden gedacht!

Feld Info:

  • Setzt man das Häkchen [IF] werden die involvierten Schnittstellen für die jeweiligen Verbindungen angegeben.
  • Setzt man das Häkchen [IP] wird für die Netzwerkgeräte die zugehörige Management-IP-Adresse angezeigt.
  • Setzt man das Häkchen [LOC] wird für die Netzwerkgeräte der zugehörige Standort angezeigt.

Feld Filter:

  • Über das erste Drop Down Menü wird der Filter gesetzt.
  • Abhängig vom gewählten Filter kann über das Textfeld die Abfrage eingegrenzt werden. Wird im Textfeld nichts eingegeben, stellt NeDi die grösstmögliche Menge dar.
  • Das untere Drop Down Menü enthält vorgefertigte Filter. Es muss darauf geachtet werden, dass diese Filter zu den restlichen Angaben in diesem Feld [Filter] passen!

Details

Wird eine Netzwerkkarte im "Geo" Modus erstellt, probiert NeDi automatisch die beste Karte zu finden. Dies ist in der Regel die Karte, welche am nächsten beim Standort liegt.

Für die "Geo" Funktion wird vorausgesetzt, dass im Modul {Topologie Loced} die Karten konfiguriert und die NeDi-spezifischen Koordinaten für die Standorte und Gebäude angegeben worden sind.

Angenommen, die Karten wurden wie in der Hilfe des Moduls {Topologie Loced} beschrieben in das ...html\log Verzeichnis kopiert und konfiguriert, wird NeDi nun z.B. beim Auswählen des Landes, die Städte auf der Landeskarte anzeigen. Wird nun eine Stadt angelickt, zeigt NeDi die entsprechende Stadtkarte mit den zugehörigen Gebäuden.

Sollten Verbindungen (z.B. zwischen Standorten) nicht angezeigt werden, können diese im Modul {Topologie Linked} manuell hinzugefügt oder angepasst werden.
