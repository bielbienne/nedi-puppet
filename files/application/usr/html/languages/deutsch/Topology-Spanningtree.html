Topologie Spanningtree

Einführung

Dieses Modul zeigt Spanningtree-Informationen eines ausgewählten Layer2-Gerätes (Switch), respektive eines Netzwerkgerätes mit Spanningtree-Funktionalität (L3 Switch mit VLAN's / Router mit Bridge Group's).

Benutzung

Im Drop Down Menü [Gerät] ein Layer2-Netzwerkgerät auswählen und auf den Button "Zeigen" klicken, so dass die vorhandenen VLAN-IDs von diesem Gerät ausgelesen werden können. Anschliessend im Drop Down Menü [Vlan] eine VLAN-ID auswählen und die Abfrage mit dem Button "Zeigen" starten. Wird im Drop Down Menü [VLAN] keine VLAN-ID ausgewählt, so werden die Spanningtree-Informationen für die VLAN-ID 1 angezeigt, welche bei den meisten Geräten ab Werk bereits vorhanden ist (Default VLAN).
Alternativ kann man noch das Häkchen [Graphen] setzen, um sich pro Schnittstelle eine Graphik zur Bandbreite und den aufgetretenen Fehlern anzeigen zu lassen. Klickt man auf eine solche Graphik, wird man zum Modul {Geräte Graphiken} weitergeleitet.

Feld Geräte Info

  • In der obersten Zeile wird links der Name und das Symbol des gewählten Netzwerkgerätes angezeigt. Klickt man auf das Netzwerksymbol, wird man zum Modul {Geräte Status} weitergeleitet und erhält detaillierte Informationen zu diesem Netzwerkgerät.
  • In der obersten Zeile wird rechts die IP-Adresse des gewählten Netzwerkgerätes angezeigt. Klickt man auf diese IP-Adresse, wird eine Telnet-Verbindung zu diesem Gerät aufgebaut.
  • Die nachfolgenden Zeilen enthalten weitere Informationen zum gewählten Netzwerkgerät.

Feld Spanningtree Info

  • Dieses Feld zeigt Spanningtree-Informationen für die gewählte VLAN-ID an.

Feld Schnittstellen

  • Das Feld [Schnittstellen | Name] zeigt auf der linken Spalte Symoble für den jeweiligen Schnittstellentyp an. In der rechten Spalte werden die Bezeichnungen der Schnittstellen aufgelistet.
  • Das Feld [Info] zeigt für die jeweiligen Schnittstellen auf den Netzwerkgeräten eingetragene Kommentare des Administrators an. Auf die Schnittstelle (Root Port), welche zum Spanningtree-Root-Gerät führt, wird mit diesem Symbol hingewiesen.
  • Das Feld [Status] zeigt in der linken Spalte den aktuellen Spanningtree-Status für die jeweiligen Schnittstellen. Die Statusinformationen werden über den Button "Zeigen" jedesmal neu vom Netzwerkgerät ausgelesen und können auf diese Weise aktualisiert werden. Die rechte Spalte gibt darüber Auskunft, ob das Spanningree-Protokoll auf dem jeweiligen Interface korrekt läuft.
  • Das Feld [Kosten] gibt Auskunft über die Pfadkosten von der jeweiligen Netzwerkschnittstelle bis zur Root-Bridge. Diese Pfadkosten werden über BPDU-Pakete (Bridge Protocol Data Units) zwischen den Spanningtree-Partnern (Switches) ermittelt. Die angezeigten Portkosten sind also immer die kumulierten Kosten aller Links bis zur Root-Bridge.
